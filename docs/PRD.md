## 1. 프로젝트 개요

### 목적

프리랜서와 소규모 에이전시가 Notion에서 작성한 견적서를 클라이언트가 브라우저에서 바로 확인하고 PDF로 저장할 수 있도록 하여, 별도 견적서 툴 없이 전문적인 견적서 전달 경험을 제공한다.

### 타겟 사용자

| 사용자 유형 | 설명 |
|------------|------|
| 서비스 제공자 | Notion을 이미 사용 중인 프리랜서, 1인 에이전시, 소규모 팀 |
| 클라이언트 | 서비스 제공자로부터 견적 링크를 받아 확인하는 최종 수신자 |

### 핵심 가치

- **Zero-friction 발송**: Notion에서 작성하면 링크 하나로 즉시 공유
- **전문적인 UI**: 깔끔하게 브랜드화된 웹 견적서로 신뢰도 향상
- **PDF 즉시 다운로드**: 클라이언트가 별도 앱 없이 바로 보관 가능

## 2. 문제 정의 및 해결책

### 현재 문제

1. 프리랜서가 Notion에서 견적서를 작성해도 클라이언트에게 공유할 때 Notion 링크를 보내면 Notion 가입이 필요하거나 레이아웃이 일관되지 않음
2. 워드/구글 독스로 견적서를 따로 만들면 Notion 데이터와 이중 관리 발생
3. PDF 수동 변환은 번거롭고 버전 관리가 어려움

### 해결책

```
Notion (데이터 원천)
        ↓  Notion API
Next.js 서버 (데이터 변환 + 렌더링)
        ↓  공유 링크
클라이언트 브라우저 (조회 + PDF 다운로드)
```

- Notion을 단일 데이터 원천(Single Source of Truth)으로 유지
- 별도 견적서 DB나 관리 UI 불필요
- 고유 토큰 기반 링크로 인증 없이도 보안 접근 제어

## 3. 사용자 스토리 및 흐름

### 서비스 제공자 (Freelancer/Agency) 흐름

```
1. Notion 데이터베이스에 견적서 페이지 생성
   ↓ (속성 입력: 클라이언트명, 금액, 항목 등)

2. 견적서 페이지에 고유 토큰 속성 값 입력 (또는 자동 생성 예정)
   ↓

3. 공유 링크 생성: https://yourdomain.com/quote/[token]
   ↓ (링크 복사)

4. 클라이언트에게 링크 전달 (이메일, 메시지 등)
   ↓

5. [완료] 클라이언트가 확인 후 승인/문의
```

### 클라이언트 흐름

```
1. 수신한 링크 클릭: /quote/[token]
   ↓

2. 견적서 페이지 로딩 (서버에서 Notion API 호출)
   ↓

   [토큰 유효] → 견적서 웹 뷰 렌더링
   [토큰 무효] → 404 / 만료 안내 페이지
   ↓

3. 견적서 내용 확인
   - 서비스 항목 및 금액 검토
   - 총액 및 세금 확인
   ↓

4. PDF 다운로드 버튼 클릭
   ↓

5. [완료] PDF 파일 로컬 저장
```

### 페이지 간 전환 조건

| 출발 페이지 | 조건 | 도착 페이지 |
|------------|------|------------|
| 견적서 조회 페이지 | 유효한 토큰 | 견적서 렌더링 완료 |
| 견적서 조회 페이지 | 유효하지 않은 토큰 | 오류 페이지 |
| 견적서 조회 페이지 | Notion API 오류 | 오류 페이지 |

## 5. 데이터 모델 (Notion 데이터베이스 구조)

### Notion 데이터베이스 속성 구조

서비스 제공자가 Notion에서 아래 속성을 가진 데이터베이스를 생성하여 사용한다.

#### 견적서 데이터베이스 (Quotes Database)

| 속성명 | Notion 속성 타입 | 설명 | 예시 |
|--------|----------------|------|------|
| `title` | Title | 견적서 제목 (페이지 이름) | "2026년 웹사이트 리뉴얼 견적" |
| `token` | Text (Rich Text) | URL 접근용 고유 토큰 | "abc123xyz" |
| `client_name` | Text (Rich Text) | 클라이언트 회사명 또는 이름 | "(주)테스트컴퍼니" |
| `client_email` | Email | 클라이언트 이메일 | "client@example.com" |
| `issue_date` | Date | 견적서 발행일 | 2026-03-31 |
| `expiry_date` | Date | 견적 유효 기간 만료일 | 2026-04-30 |
| `currency` | Select | 통화 단위 | KRW, USD |
| `tax_rate` | Number | 세율 (%) | 10 |
| `notes` | Text (Rich Text) | 하단 비고 및 안내 문구 | "계약금 50% 선입금 후 착수" |
| `status` | Select | 견적서 상태 | 초안, 발송됨, 만료됨 |

#### 견적 항목 (Line Items) - Notion 페이지 본문 블록 활용

견적 항목은 Notion 페이지 본문의 **테이블 블록** 또는 **하위 데이터베이스**로 관리한다.

**옵션 A: 페이지 본문 테이블 블록 (권장 - 단순)**

| 열 이름 | 설명 | 타입 |
|--------|------|------|
| `항목명` | 서비스/상품명 | 텍스트 |
| `수량` | 수량 | 숫자 |
| `단가` | 단위 금액 | 숫자 |
| `금액` | 수량 x 단가 (계산) | 숫자 |
| `비고` | 항목별 메모 | 텍스트 |

**옵션 B: 하위 데이터베이스 (고급 - 재사용성 높음)**

| 속성명 | Notion 속성 타입 | 설명 |
|--------|----------------|------|
| `item_name` | Title | 항목명 |
| `quantity` | Number | 수량 |
| `unit_price` | Number | 단가 |
| `quote_id` | Relation | 상위 견적서 연결 |

### 서비스 제공자 정보 (브랜딩용)

Notion 데이터베이스와 별도로 환경변수 또는 설정 파일로 관리한다.

| 필드명 | 설명 |
|--------|------|
| `PROVIDER_NAME` | 회사명 또는 프리랜서 이름 |
| `PROVIDER_EMAIL` | 대표 연락 이메일 |
| `PROVIDER_PHONE` | 연락처 (선택) |
| `PROVIDER_ADDRESS` | 주소 (선택) |
| `PROVIDER_LOGO_URL` | 로고 이미지 URL (선택) |

## 6. API 설계

### 6-1. 내부 API (Next.js Route Handler)

#### GET /api/quote/[token]

견적서 토큰으로 Notion에서 데이터를 조회하여 반환한다.

**요청**

```
GET /api/quote/abc123xyz
```

**응답 - 성공 (200)**

```json
{
  "id": "notion-page-id",
  "title": "2026년 웹사이트 리뉴얼 견적",
  "client": {
    "name": "(주)테스트컴퍼니",
    "email": "client@example.com"
  },
  "provider": {
    "name": "홍길동 디자인",
    "email": "contact@provider.com",
    "phone": "010-0000-0000",
    "logo_url": "https://..."
  },
  "issue_date": "2026-03-31",
  "expiry_date": "2026-04-30",
  "currency": "KRW",
  "tax_rate": 10,
  "line_items": [
    {
      "name": "UI/UX 디자인",
      "quantity": 1,
      "unit_price": 2000000,
      "amount": 2000000,
      "note": "와이어프레임 포함"
    }
  ],
  "subtotal": 2000000,
  "tax_amount": 200000,
  "total": 2200000,
  "notes": "계약금 50% 선입금 후 착수",
  "status": "발송됨"
}
```

**응답 - 토큰 없음 (404)**

```json
{
  "error": "QUOTE_NOT_FOUND",
  "message": "유효하지 않은 견적서 링크입니다."
}
```

**응답 - 만료됨 (410)**

```json
{
  "error": "QUOTE_EXPIRED",
  "message": "견적서 유효 기간이 만료되었습니다.",
  "expiry_date": "2026-04-30"
}
```

**응답 - 서버 오류 (500)**

```json
{
  "error": "NOTION_API_ERROR",
  "message": "데이터를 불러오는 중 오류가 발생했습니다."
}
```

### 6-2. Notion API 연동

#### 사전 준비

1. Notion Integration 생성: https://www.notion.so/my-integrations
2. Integration Token 발급 (환경변수 `NOTION_API_KEY`로 저장)
3. 견적서 데이터베이스 페이지에서 Integration 연결 허용
4. 데이터베이스 ID 복사 (환경변수 `NOTION_DATABASE_ID`로 저장)

#### 견적서 토큰으로 페이지 조회

```
POST https://api.notion.com/v1/databases/{database_id}/query
Authorization: Bearer {NOTION_API_KEY}
Notion-Version: 2022-06-28
Content-Type: application/json

{
  "filter": {
    "property": "token",
    "rich_text": {
      "equals": "{token}"
    }
  }
}
```

#### 페이지 블록(항목) 조회

```
GET https://api.notion.com/v1/blocks/{page_id}/children
Authorization: Bearer {NOTION_API_KEY}
Notion-Version: 2022-06-28
```

#### 환경변수 목록

| 변수명 | 설명 | 필수 여부 |
|--------|------|----------|
| `NOTION_API_KEY` | Notion Integration Secret Token | 필수 |
| `NOTION_DATABASE_ID` | 견적서 데이터베이스 ID | 필수 |
| `PROVIDER_NAME` | 서비스 제공자 이름/회사명 | 필수 |
| `PROVIDER_EMAIL` | 서비스 제공자 이메일 | 필수 |
| `PROVIDER_PHONE` | 서비스 제공자 연락처 | 선택 |
| `PROVIDER_ADDRESS` | 서비스 제공자 주소 | 선택 |
| `PROVIDER_LOGO_URL` | 로고 이미지 URL | 선택 |

## 7. 비기능 요구사항

### 7-1. 보안

| 항목 | 요구사항 |
|------|---------|
| **링크 기반 접근 제어** | 견적서 조회는 URL 토큰으로만 가능. 토큰 없이 목록 조회 불가 |
| **토큰 형식** | 최소 16자 이상의 무작위 영숫자 문자열 (UUID v4 권장) |
| **Notion API 키 보호** | `NOTION_API_KEY`는 서버 사이드 환경변수로만 사용. 클라이언트 번들에 노출 금지 |
| **서버 사이드 처리** | 모든 Notion API 호출은 Next.js Route Handler에서 수행 |
| **열거 공격 방지** | 잘못된 토큰에 대해 일관된 404 응답 반환 (존재 여부 추론 불가) |
| **HTTPS 강제** | 프로덕션 환경에서 HTTP → HTTPS 리디렉션 필수 |

### 7-2. 성능

| 항목 | 목표 |
|------|------|
| **초기 로딩** | 견적서 페이지 LCP 3초 이내 |
| **Notion API 응답** | 타임아웃 설정 10초, 초과 시 오류 처리 |
| **캐싱 전략** | 동일 토큰 재요청 시 Next.js `revalidate` 또는 `unstable_cache`로 60초 캐싱 |
| **PDF 생성** | 브라우저 기본 Print API 사용 (서버 PDF 생성 라이브러리 불필요) |

### 7-3. 접근성 및 사용성

| 항목 | 요구사항 |
|------|---------|
| **반응형 디자인** | 모바일(360px), 태블릿(768px), 데스크탑(1280px) 지원 |
| **인쇄 스타일** | `@media print` CSS로 헤더/버튼 숨김, 여백 최적화 |
| **오류 메시지** | 기술 용어 없이 사용자 친화적 한국어 안내 |
| **로딩 피드백** | API 호출 중 스켈레톤 UI 또는 로딩 스피너 표시 |

### 7-4. 기술 제약사항

| 항목 | 결정 |
|------|------|
| **런타임** | Next.js 16 App Router, TypeScript |
| **PDF 방식** | `window.print()` + 인쇄용 CSS (추가 라이브러리 없음) |
| **인증** | 없음 (MVP 범위 외). 토큰 기반 링크 접근만 사용 |
| **데이터베이스** | 없음 (MVP 범위 외). Notion이 유일한 데이터 저장소 |
| **배포** | Vercel (Next.js 최적화 환경) |

```
견적서 조회 페이지 (/quote/[token])
├── 견적서 헤더 (제공자 로고, 이름, 연락처)
├── 견적서 메타 정보 (발행일, 유효기간, 클라이언트명)

오류 페이지 (/quote/[token] - 오류 상태)
└── 오류 안내 메시지 (F010)
```

## 페이지별 상세 기능

### 견적서 조회 페이지

> **구현 기능:** `F001`, `F002`, `F003`, `F004`, `F005`, `F011` | **인증:** 토큰 기반 (인증 없음)

| 항목 | 내용 |
|------|------|
| **역할** | 서비스의 핵심 페이지. Notion 데이터를 시각적으로 렌더링하고 PDF 다운로드를 제공 |
| **진입 경로** | 서비스 제공자가 클라이언트에게 전달한 공유 링크 직접 접근 (/quote/[token]) |
| **사용자 행동** | 견적서 내용 열람, 항목 및 금액 확인, PDF 다운로드 버튼 클릭 |
| **주요 기능** | 토큰 유효성 검증 (F004), Notion API 데이터 조회 (F001), 견적서 레이아웃 렌더링 (F002), 항목/금액 계산 표시 (F005), 로딩 스피너 표시 (F011), **PDF 다운로드** 버튼 (F003) |
| **다음 이동** | 성공 → 견적서 렌더링 완료, 실패 → 오류 페이지로 이동 |

---

### 오류 페이지

> **구현 기능:** `F010` | **인증:** 없음

| 항목 | 내용 |
|------|------|
| **역할** | 잘못된 토큰, 만료된 링크, Notion API 오류 발생 시 사용자에게 안내 |
| **진입 경로** | 견적서 조회 페이지에서 토큰 검증 실패 또는 API 오류 시 자동 전환 |
| **사용자 행동** | 오류 메시지 확인 후 서비스 제공자에게 문의 |
| **주요 기능** | 오류 유형별 안내 메시지 (유효하지 않은 링크 / 만료된 견적 / 서버 오류), 서비스 제공자 연락처 표시 |
| **다음 이동** | 해당 없음 (종료 페이지) |
